Confirmed — `cpvGroupRange` builds a half-open `[group, hi)` range by incrementing the last digit's char code, so a group ending in '9' (e.g. `45239`) yields `hi = "4524:"`, which still correctly bounds every `4523900000`-style prefix while excluding the next group. Inputs are validated to exactly 5 digits before reaching this function (`cpvGroupSelection` in filters.ts), and the range bounds are always bind-parameterized (`t.cpv_code >= ? AND t.cpv_code < ?`), so there's no injection surface here. No code change needed.
