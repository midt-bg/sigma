# Публични данни и endpoint-и (за разработчици)

СИГМА е портал само за четене върху българските обществени поръчки. Освен
HTML-страниците, всеки списък и всеки договор се отдават и в **машинно четим**
вид, така че да можете да строите върху данните (приложения, анализи, визуализации)
без да скрейпвате HTML. Тази страница описва какво е налично днес.

> **Източник и лиценз.** Данните идват от официалния регистър ЦАИС ЕОП / АОП
> (`storage.eop.bg`), отворени данни, **CC-BY 4.0**. Атрибутирайте „АОП / ЦАИС ЕОП"
> и по желание СИГМА. СИГМА нормализира и обогатява; сумите се показват както са
> в източника, с едно изключение: при малък набор редове с явно сгрешена сума
> (маркирани `value_suspect`) `amount_eur` показва прогнозната стойност на
> процедурата вместо сгрешената сума. Всеки запис сочи обратно към деня-източник
> в `storage.eop.bg`.

## Модел на данните

Три същности и връзките помежду им:

- **Институция** (възложител) — ключ ЕИК.
- **Компания** (изпълнител) — ключ ЕИК.
- **Договор** — сключената сделка, с проследимост до източника.

плюс **парични потоци** институция → компания. Паричните стойности са в **евро**
(`amount_eur`); `null` означава реално липсваща/непотвърдена стойност, никога
измислена.

## Машинно четими endpoint-и

> **Лични данни (ЕТ).** CSV експортите и JSON-ът на договор днес включват
> регистрови идентификатори (ЕИК), които при едноличните търговци (ЕТ) са
> идентификатори на физически лица, без да разграничават ЕТ от юридически лица.
> HTML профилите на разпознатите ЕТ са `noindex`, а `/sitemap-companies.xml`
> пропуска профили с ЕТ-име (евристика по име — не хваща всички случаи).
> Политиката по въпроса се уточнява в issue #173 — дотогава bulk потребителите
> трябва да третират тези полета като лични данни по смисъла на GDPR.

### CSV експорт на списъците

| Endpoint | Съдържание |
|---|---|
| `GET /contracts.csv` | договори |
| `GET /companies.csv` | компании |
| `GET /authorities.csv` | институции |

`Content-Type: text/csv; charset=utf-8`, стриймнат. **Уважават същите филтри**
като HTML списъка — добавете query параметрите по-долу и сваляте точно показания
срез (нефилтрираният пълен експорт е кеширан и бърз).

### JSON на договор

`GET /contracts/{id}.json` → пълният запис на договора като
`application/json; charset=utf-8` (`{ "error": "not_found" }` + 404, ако липсва).
Това е единственият per-entity JSON днес; за институции/компании ползвайте CSV
списъците или HTML профилите.

### Sitemap-и

`GET /sitemap.xml` (индекс) + `/sitemap-pages.xml`, `/sitemap-authorities.xml`,
`/sitemap-companies.xml`, `/sitemap-contracts.xml`, и `/robots.txt`. Удобни за
програмно изброяване на URL-ите (вкл. всеки договор; `/sitemap-companies.xml`
пропуска профилите с ЕТ-име — вж. бележката за личните данни по-горе).

## Query грамата на филтрите

Параметрите живеят в адреса (всяка комбинация има постоянен, споделяем линк) и са
общи за HTML списъка и неговия `.csv`:

| Списък | Параметри |
|---|---|
| `/contracts(.csv)` | `year`, `sector`, `procedure`, `value`, `eu`, `authority`, `bidder`, `q`, `bids`, `sort` |
| `/companies(.csv)` | `kind`, `count`, `sector`, `year`, `eu`, `q`, `sort` |
| `/authorities(.csv)` | `type`, `sector`, `year`, `eu`, `q`, `sort` |

Многозначните параметри се повтарят или се изброяват със запетая
(`?year=2025&year=2024` или `?year=2025,2024`). HTML списъците са странирани с
keyset курсор (`cursor`); CSV-то връща целия филтриран срез наведнъж.

## Какво (още) НЯМА

За да остане порталът бърз, евтин и устойчив, **няма** отворен SQL/заявков
endpoint и **няма** обща REST заявка отвъд изброеното по-горе — данните се отдават
през фиксирани, кеширани, анонимни endpoint-и. Endpoint-ите са rate-limited.
Няма write/auth API (всичко е само за четене).

За **bulk** работа: изброете през sitemap-ите и/или свалете трите CSV-та; за
суровите дневни файлове отидете директно на `storage.eop.bg` (СИГМА не държи копие).
