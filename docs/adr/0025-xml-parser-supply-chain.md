# ADR-0025 — Верига на доставки за XML парсера (fast-xml-parser)

- **Статус:** Прието
- **Дата:** 2026-07-10
- **Обхват:** ETL парсване на декларациите (`scripts/cacbg`), слой „свързани лица"

## Контекст

Парсването на декларациите от Публичния регистър на Сметната палата (ADR-0013) ползва
`fast-xml-parser`. Версия `^5.9.3` въвежда ново модуларизирано транзитивно дърво
(`is-unsafe`, `xml-naming`, `path-expression-matcher`, `@nodable/entities`, `fast-xml-builder`,
`anynum`). XML от външен, недоверен източник е класическа входна точка (XXE, entity-expansion),
а по-широкото дърво увеличава повърхността за компрометиране по веригата на доставки. Caret
диапазонът (`^`) допуска мълчалив дрейф на второстепенни версии.

## Решение

Запазваме `fast-xml-parser` за парсването. Транзитивното дърво е одитирано: integrity в
регистъра съвпада, няма install-скриптове, кодът е само в XML домейна. XXE е двойно защитен
(изключени външни ентитита + тест). Дисциплина по веригата на доставки:

- `pnpm install --frozen-lockfile` във **всеки** CI workflow (`ci`, `deploy`, `scripts-test`,
  `related-persons-data`) — версиите се разрешават само от заключения lockfile, никога наново.
- Обновяване на парсера или дървото му минава през ревю: повторен одит (integrity,
  install-скриптове, домейн на кода) преди merge.

## Последствия

- Възпроизводими инсталации; дрейф по caret диапазона не може да влезе мълчаливо в CI/прод.
- Цена: обновяване на `fast-xml-parser` изисква ръчен повторен одит на дървото — не е автоматично.
- Follow-up: нова зависимост с install-скриптове налага препоценка на тази позиция.
